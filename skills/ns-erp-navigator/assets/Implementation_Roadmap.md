# Implementation Roadmap Templates

Choose the roadmap that matches the confirmed **Project Tier**.

---

## Tier 1: Full Implementation (6 Months)

| Month | Phase | Key Milestones |
| :--- | :--- | :--- |
| **1** | **Discovery** | BRD Signed Off, Project Charter Approved. |
| **2** | **Design** | SDDs Approved, Fit-Gap Analysis Complete. |
| **3** | **Build** | Core Configuration, Custom Scripts/Workflows. |
| **4** | **Data & UAT** | Master Data Loaded, UAT Scripts Written. |
| **5** | **UAT & Training** | User Acceptance Testing, Training Delivered. |
| **6** | **Go-Live** | Cutover Weekend, Hypercare Starts. |

---

## Tier 2: Module / Refactoring (6 Weeks)

| Week | Phase | Key Milestones |
| :--- | :--- | :--- |
| **1** | **Discovery/Design** | Scoping complete, Functional Spec approved. |
| **2-3** | **Build** | Module configuration & testing. |
| **4** | **Data/Testing** | Data updates, Internal Smoke Testing. |
| **5** | **UAT** | Targeted User Testing & Sign-off. |
| **6** | **Go-Live** | Deployment & Hypercare. |

---

## Tier 3: Tiny Enhancement (2 Weeks)

| Day | Phase | Key Milestones |
| :--- | :--- | :--- |
| **1** | **Discovery** | Requirements confirmed via email/chat. |
| **2** | **Design/Build** | Simplified Spec written & configuration starts. |
| **3-7** | **Build/Test** | Development and Internal QA. |
| **8-9** | **UAT** | User verification in Sandbox. |
| **10** | **Go-Live** | Production deployment & Smoke Test. |

---

## Peak Season Blackouts

Avoid Go-Live during these periods unless mission-critical:
- **December 15 – January 15:** Year-end close & holiday freeze.
- **Q1 (Jan – March):** Tax season for many finance teams.
- **Project-Specific:** Major marketing launches or seasonal peaks.
